1. このページの目的

Single File Demo

AIが出力した文章を、読みやすく整形して第三者へ共有する場合、 「HTMLとして1枚の静的ページにする」方法はかなり便利です。 GitHub Pages、Netlify Drop、Cloudflare Pages、S3などに置けば、URLだけで共有できます。

3. おすすめ運用フロー

タイムライン

Step 1：AIに本文を作らせる

まずは普通に文章、分析、まとめ、比較表、FAQなどを作ります。この段階ではMarkdownでも構いません。

Step 2：HTML化を依頼する

「CSSとJavaScript込みで、単体HTMLにしてください」と依頼します。今回のページがまさにその例です。

Step 3：ファイル名を決める

posts/2026-05-09-ai-share.html のように、日付入りの名前にすると後から管理しやすくなります。

Step 4：GitHub Pagesなどに配置する

GitHubのWeb画面で新規ファイルを作成し、HTMLを貼ってコミットします。

Step 5：URLを共有する

公開URLを相手に送ります。公開前に個人情報や非公開情報が入っていないか確認しましょう。

注意： 静的HTMLにした時点で、ページは「URLを知っている人が閲覧できるもの」と考えてください。秘密情報の共有には向いていません。

7. FAQ

details / summary
このHTMLは本当に1ファイルだけで動きますか？
はい。CSSは<style>内、JavaScriptは<script>内に含まれています。 外部画像、外部フォント、CDN、ライブラリは使っていません。
スマホだけでGitHub Pagesに公開できますか？
可能です。GitHubのWeb画面からファイル作成・編集・コミットができます。 ただし、長いHTMLはスマホのテキスト編集がやや大変なので、単発ならNetlify Dropのほうが楽な場合もあります。
S3のほうが安いならS3を使うべきですか？
料金だけならS3は非常に安いです。ただし、公開設定や権限、HTTPS、独自ドメインなどを考えると、 気軽な共有用途ではGitHub PagesやNetlifyのほうが扱いやすいことが多いです。
検索エンジンに見つかりたくない場合は？
完全に見つからない保証はできません。URLを知っている人だけに見せたい場合でも、 静的公開ページは「半公開」くらいに考えたほうが安全です。 本当に限定したい場合は、認証やパスワード付きのサービスを検討してください。
