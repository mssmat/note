Gemini CLIをRaspberry Piにインストールして使う方法

GitHub - google-gemini/gemini-cli: An open-source AI agent that brings the power of Gemini directly into your terminal.

An open-source AI agent that brings the power of Gemini directly into your...

github.com

ゴール

iPhoneから自宅のRaspberry PiにSSHで接続して使う

インストール

bash 
# 大前提としてNode.jsとnpmがインストールされていること
sudo npm install -g @google/gemini-cligemini

立ち上げ

bash 
gemini

認証

gemini-cli/docs/cli/authentication.md at main · google-gemini/gemini-cli · GitHub

An open-source AI agent that brings the power of Gemini directly into your...

github.com

認証はGemini CLIを立ち上げた状態で、以下のコマンドで行う(初回はすぐに認証を求められる)

bash 
/auth

「Login with Google」を選択するのが一般的で、ブラウザが立ち上がり、そこでGoogleの個人アカウントで認証すると使えるようになる
基本的に初回のみ認証が必要で、一度認証すれば、SSHなどを使って、他の端末からも利用できる

SSHで接続して使う様子

Google announces Gemini CLI: your open-source AI agent

Free and open source, Gemini CLI brings Gemini directly into developers’...

blog.google

Gemini CLIの無料枠は以下の通り(超過後もAPIキーの利用などで使えるらしい)

Gemini CLI offers the industry's largest usage allowance at 60 model requests per minute and 1,000 model requests per day at no charge

認証を削除

恐らく以下に認証後に生成・更新される各種ファイルが入っている

bash 
ls /home/pi/.gemini
# oauth_creds.json  settings.json  tmp  user_id

これらのうち「oauth_creds.json / settings.json / user_id」を削除すれば初期化され、再度認証が必要となるようだ

bash 
cd /home/pi/.gemini
sudo rm oauth_creds.json
sudo rm settings.json
sudo rm user_id

🤔【TypeScriptベース】opencodeを試してみた

作成日時2025-06-21 10:11 TagAI, GitHub

先日Claude Codeの代替品としてOpenCodeという”TUI”コーディングツールを紹介しましたが、現在Xではもう1つのopencodeが話題になっています。
こちらはClaude ProやMaxプランに対応しているそうで、それで話題になっているみたいですね。
このopencodeはTypeScript版だそうです。

GitHub - sst/opencode: AI coding agent, built for the terminal.

AI coding agent, built for the terminal. Contribute to sst/opencode development...

github.com

同じ名前のが2つもあって混乱しますが、何やら揉めた様子..

The other confusingly named repo has no relation to this one. You can read the story behind it here.

Go版のOpenCodeを入れている場合、競合するかも知れないので削除しておいた方がよいかも知れません。

いつものようにRaspberry Piでやったので共有します。

実際に使っている様子(iPhone)

GoベースのOpenCodeのアンインストール

  • インストールの場所を確認
bash 
which opencode
  • アンインストール
bash 
sudo rm /home/pi/.opencode/bin/opencode
  • 設定ファイルを削除
bash 
rm -rf ~/.config/opencode
  • モデルとAPIキー設定を削除
bash 
rm ~/.opencode.json
  • 他に関連ファイルがないか確認しあれば削除する
bash 
sudo find / -iname "*opencode*" 2>/dev/null

TypeScriptベースのopencodeの新規インストール

  • 本来このコマンドだが、今現在最新のアセットにLinux arm用のファイルがなくダウンロードできなかった(v0.1.115)
bash 
curl -fsSL https://opencode.ai/install | bash
  • そのため一つ古いバージョンのファイルを手動でインストールする
  • ダウンロード
bash 
wget "https://github.com/sst/opencode/releases/download/v0.1.114/opencode-linux-arm64.zip"
  • 解凍
bash 
unzip opencode-linux-arm64.zip
  • 設置場所の作成
bash 
mkdir -p $HOME/.opencode/bin
  • 実行ファイルの移動
bash 
mv opencode $HOME/.opencode/bin/
  • 設定を追記
bash 
echo 'export PATH="$HOME/.opencode/bin:$PATH"' >> ~/.bashrc
  • OpenRouther APIキーの保存
bash 
echo 'export OPENROUTER_API_KEY="ここにOpenRouterのAPIキー"' >> ~/.bashrc
  • 保存したくない場合
bash 
export OPENROUTER_API_KEY="ここにOpenRouterのAPIキー"
  • 設定を再読み込み
bash 
source ~/.bashrc
  • 設定ファイル用のディレクトリを作成する
bash 
mkdir -p ~/.config/opencode
  • 設定ファイルを編集
bash 
nano ~/.config/opencode/config.json
  • JSON(OpenRouterのGemini 2.5 Flashを使う場合)
json 
{
  "$schema": "https://opencode.ai/config.json",
  "theme": "opencode",
  "autoupdate": true,
  "provider": {
    "openrouter": {
      "npm": "@openrouter/ai-sdk-provider",
      "name": "OpenRouter",
      "options": {},
      "models": {
        "google/gemini-2.5-flash": {
          "name": "Gemini 2.5 Flash"
        }
      }
    }
  },
  "model": "openrouter/google/gemini-2.5-flash"
}
  • 作業用ディレクトリを作成する
bash 
mkdir ~/opencode
  • 実行
bash 
cd ~/opencode
opencode
