概要

OpenCutは、人気の動画編集アプリ**CapCut**のオープンソース版代替を目指す、無料の動画編集ツールです。
現在の主な利用形態は、自身のPC（ローカル環境）に開発環境をセットアップし、Webブラウザ経由で操作するものとなります。

  • 特徴:
    • 無料で利用できるオープンソースプロジェクト
    • 自身のPCにセットアップし、Webブラウザ (localhost) で操作
    • インストールと実行には git と npm の知識が必要
    • 開発初期段階であり、今後の機能追加や改善が期待される

    1\. インストール (セットアップ)

    OpenCutを自分のPCで起動し、ブラウザで使えるようにするまでの手順です。

    1. ソースコードをダウンロード ターミナルを開き、任意の場所にOpenCutのソースコードをダウンロード（クローン）します。
      shell 
      git clone <https://github.com/OpenCut-app/OpenCut.git>
    2. プロジェクトフォルダへ移動 ダウンロードしたフォルダの中にある、Webアプリケーション用のディレクトリに移動します。
      shell 
      cd OpenCut/apps/web
    3. 依存関係をインストール プロジェクトの実行に必要なプログラム（依存関係）をインストールします。
      shell 
      npm install
    4. 開発サーバーを起動 ローカルサーバーを起動します。
      shell 
      npm run dev
    5. ブラウザでアクセス ターミナルにready in ...と表示されたら、Webブラウザを開き、以下のアドレスにアクセスします。 http://localhost:3000

    2\. アップデート

    プロジェクトを最新の状態に更新する手順です。

    1. プロジェクトフォルダへ移動 ターミナルで、OpenCut/apps/web ディレクトリにいることを確認してください。
      shell 
      # 例
cd path/to/your/OpenCut/apps/web
    2. ソースコードを最新化gitコマンドで、GitHub上の最新のソースコードを取得します。
      shell 
      git pull
    3. 依存関係を更新 ソースコードの変更に合わせて、必要なプログラムを更新します。
      shell 
      npm install

      完了後、再度 npm run dev で起動すれば、最新版のOpenCutを利用できます。

    3\. アンインストール (削除)

    PCからOpenCutを完全に削除する手順です。

    • プロジェクトフォルダを丸ごと削除するだけで完了です。

    npmによるローカルインストールでは、PCの他の領域にファイルが書き込まれることはありません。そのため、最初にgit cloneで作成したOpenCutフォルダを削除すれば、アンインストールは完了します。

    🤔【TypeScriptベース】opencodeを試してみた

    作成日時2025-06-21 10:11 TagAI, GitHub

    先日Claude Codeの代替品としてOpenCodeという”TUI”コーディングツールを紹介しましたが、現在Xではもう1つのopencodeが話題になっています。
    こちらはClaude ProやMaxプランに対応しているそうで、それで話題になっているみたいですね。
    このopencodeはTypeScript版だそうです。

    GitHub - sst/opencode: AI coding agent, built for the terminal.

    AI coding agent, built for the terminal. Contribute to sst/opencode development...

    github.com

    同じ名前のが2つもあって混乱しますが、何やら揉めた様子..

    The other confusingly named repo has no relation to this one. You can read the story behind it here.

    Go版のOpenCodeを入れている場合、競合するかも知れないので削除しておいた方がよいかも知れません。

    いつものようにRaspberry Piでやったので共有します。

    実際に使っている様子(iPhone)

    GoベースのOpenCodeのアンインストール

    • インストールの場所を確認
    bash 
    which opencode
    • アンインストール
    bash 
    sudo rm /home/pi/.opencode/bin/opencode
    • 設定ファイルを削除
    bash 
    rm -rf ~/.config/opencode
    • モデルとAPIキー設定を削除
    bash 
    rm ~/.opencode.json
    • 他に関連ファイルがないか確認しあれば削除する
    bash 
    sudo find / -iname "*opencode*" 2>/dev/null

    TypeScriptベースのopencodeの新規インストール

    • 本来このコマンドだが、今現在最新のアセットにLinux arm用のファイルがなくダウンロードできなかった(v0.1.115)
    bash 
    curl -fsSL https://opencode.ai/install | bash
    • そのため一つ古いバージョンのファイルを手動でインストールする
    • ダウンロード
    bash 
    wget "https://github.com/sst/opencode/releases/download/v0.1.114/opencode-linux-arm64.zip"
    • 解凍
    bash 
    unzip opencode-linux-arm64.zip
    • 設置場所の作成
    bash 
    mkdir -p $HOME/.opencode/bin
    • 実行ファイルの移動
    bash 
    mv opencode $HOME/.opencode/bin/
    • 設定を追記
    bash 
    echo 'export PATH="$HOME/.opencode/bin:$PATH"' >> ~/.bashrc
    • OpenRouther APIキーの保存
    bash 
    echo 'export OPENROUTER_API_KEY="ここにOpenRouterのAPIキー"' >> ~/.bashrc
    • 保存したくない場合
    bash 
    export OPENROUTER_API_KEY="ここにOpenRouterのAPIキー"
    • 設定を再読み込み
    bash 
    source ~/.bashrc
    • 設定ファイル用のディレクトリを作成する
    bash 
    mkdir -p ~/.config/opencode
    • 設定ファイルを編集
    bash 
    nano ~/.config/opencode/config.json
    • JSON(OpenRouterのGemini 2.5 Flashを使う場合)
    json 
    {
  "$schema": "https://opencode.ai/config.json",
  "theme": "opencode",
  "autoupdate": true,
  "provider": {
    "openrouter": {
      "npm": "@openrouter/ai-sdk-provider",
      "name": "OpenRouter",
      "options": {},
      "models": {
        "google/gemini-2.5-flash": {
          "name": "Gemini 2.5 Flash"
        }
      }
    }
  },
  "model": "openrouter/google/gemini-2.5-flash"
}
    • 作業用ディレクトリを作成する
    bash 
    mkdir ~/opencode
    • 実行
    bash 
    cd ~/opencode
opencode
