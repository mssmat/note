🤔【TypeScriptベース】opencodeを試してみた

作成日時2025-06-21 10:11 TagAI, GitHub

先日Claude Codeの代替品としてOpenCodeという”TUI”コーディングツールを紹介しましたが、現在Xではもう1つのopencodeが話題になっています。
こちらはClaude ProやMaxプランに対応しているそうで、それで話題になっているみたいですね。
このopencodeはTypeScript版だそうです。

GitHub - sst/opencode: AI coding agent, built for the terminal.

AI coding agent, built for the terminal. Contribute to sst/opencode development...

github.com

同じ名前のが2つもあって混乱しますが、何やら揉めた様子..

The other confusingly named repo has no relation to this one. You can read the story behind it here.

Go版のOpenCodeを入れている場合、競合するかも知れないので削除しておいた方がよいかも知れません。

いつものようにRaspberry Piでやったので共有します。

実際に使っている様子(iPhone)

GoベースのOpenCodeのアンインストール

  • インストールの場所を確認
bash 
which opencode
  • アンインストール
bash 
sudo rm /home/pi/.opencode/bin/opencode
  • 設定ファイルを削除
bash 
rm -rf ~/.config/opencode
  • モデルとAPIキー設定を削除
bash 
rm ~/.opencode.json
  • 他に関連ファイルがないか確認しあれば削除する
bash 
sudo find / -iname "*opencode*" 2>/dev/null

TypeScriptベースのopencodeの新規インストール

  • 本来このコマンドだが、今現在最新のアセットにLinux arm用のファイルがなくダウンロードできなかった(v0.1.115)
bash 
curl -fsSL https://opencode.ai/install | bash
  • そのため一つ古いバージョンのファイルを手動でインストールする
  • ダウンロード
bash 
wget "https://github.com/sst/opencode/releases/download/v0.1.114/opencode-linux-arm64.zip"
  • 解凍
bash 
unzip opencode-linux-arm64.zip
  • 設置場所の作成
bash 
mkdir -p $HOME/.opencode/bin
  • 実行ファイルの移動
bash 
mv opencode $HOME/.opencode/bin/
  • 設定を追記
bash 
echo 'export PATH="$HOME/.opencode/bin:$PATH"' >> ~/.bashrc
  • OpenRouther APIキーの保存
bash 
echo 'export OPENROUTER_API_KEY="ここにOpenRouterのAPIキー"' >> ~/.bashrc
  • 保存したくない場合
bash 
export OPENROUTER_API_KEY="ここにOpenRouterのAPIキー"
  • 設定を再読み込み
bash 
source ~/.bashrc
  • 設定ファイル用のディレクトリを作成する
bash 
mkdir -p ~/.config/opencode
  • 設定ファイルを編集
bash 
nano ~/.config/opencode/config.json
  • JSON(OpenRouterのGemini 2.5 Flashを使う場合)
json 
{
  "$schema": "https://opencode.ai/config.json",
  "theme": "opencode",
  "autoupdate": true,
  "provider": {
    "openrouter": {
      "npm": "@openrouter/ai-sdk-provider",
      "name": "OpenRouter",
      "options": {},
      "models": {
        "google/gemini-2.5-flash": {
          "name": "Gemini 2.5 Flash"
        }
      }
    }
  },
  "model": "openrouter/google/gemini-2.5-flash"
}
  • 作業用ディレクトリを作成する
bash 
mkdir ~/opencode
  • 実行
bash 
cd ~/opencode
opencode

Claude Codeの代替品「OpenCode」をRaspberry Piにインストールして使ってみた

作成日時2025-06-09 18:43 TagAI, GitHub
💡
ゴールは、「OpenCode」を「Raspberry Pi」にインストールし、「OpenRouter」の「Gemini 2.5 Flash」を使うこと。

実際に使っている様子(PC/iPhone)

これは何? OpenCodeとは?

OpenCodeは、ターミナルでAIアシスタンスを実現するGoベースのCLIアプリケーションです。様々なAIモデルと対話するための”TUI”を提供し、コーディング作業やデバッグなどを支援します。
巷では、Claude Codeの”代替品”などと呼ばれています(他に似たものとして、OpenAIのオープンソース、Codex CLIがありますね)。
Claude Codeを使いたいけど、Claude以外のモデルを使いたい..というような要望に応えます。
GitHub - opencode-ai/opencode: A powerful AI coding agent. Built for the terminal.

A powerful AI coding agent. Built for the terminal. - opencode-ai/opencode

github.com

手順

  • 本体をインストール
bash 
curl -fsSL https://raw.githubusercontent.com/opencode-ai/opencode/refs/heads/main/install | bash
  • 関連ソフトをインストール
bash 
sudo apt update
sudo apt install ripgrep fzf
  • 設定ファイルを作成
bash 
nano ~/.opencode.json
